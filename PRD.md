## プロジェクト概要
- my-reportページを中心にTikTokアカウントの自動取得データと手入力データを統合し、レポート出力までを一貫して提供する。

## ゴール
- my-reportページを完成させ、ユーザーが自アカウントの最新指標と推移を確認・出力できる状態にする。

## 機能要件
1. TikTok API（Login Kit: user.info.basic, user.info.stats, video.list）と連携し、ユーザーが自アカウントを接続して日次で動画・エンゲージメントデータを取得し、現時点の値と推移を表示できること。
2. 月初〜月末の期間を指定して、取得データをCSV形式で出力できること。
3. 各動画の2秒視聴率・6秒視聴率・視聴完了率を手入力またはCSVインポートで登録できること。
4. API取得データと手入力データを組み合わせ、PowerPoint形式のレポートを生成できること。
5. 生成したPowerPointのフッターにユーザー企業のロゴを配置でき、簡易なカスタマイズにも対応できること。

## データ同期フロー
1. 初回連携時は `/api/auth/tiktok/complete` が成功した直後にバックグラウンド同期タスクを起動し、TikTok API (`video.list`, `user.info.*`) から取得したデータを `users_videos` と `users_video_daily_metrics_new` に保存する。
2. 連携済みユーザーは Cloud Scheduler（もしくは等価のジョブランナー）から日次バッチを起動し、リフレッシュトークンを用いてアクセストークンを更新した上で同様の同期処理を行う。
3. 連携解除リクエストを受けたら `users_tiktok_accounts` / `tiktok_tokens` を削除し、日次バッチ対象から除外する。参照テーブルも必要に応じてクリーニングする。
