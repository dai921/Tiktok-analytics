# account_follower_history テーブル

## 概要
TikTokアカウントのフォロワー数を日次で記録する履歴テーブル。スクレイピング等で取得した表示テキストと、正規化済みの数値の両方を保持する。

## テーブル定義

| 列名 | データ型 | NULL | デフォルト | インデックス | 説明 |
|-----|---------|------|----------|------------|------|
| id | BIGINT UNSIGNED | NO | AUTO_INCREMENT | PRIMARY | レコードの一意識別子（主キー） |
| account_id | INT | NO |  | | 対象アカウントのID（`account_list.id`） |
| collection_date | DATE | NO |  | | データ収集日（2日1レコード） |
| follower_text | VARCHAR(32) | YES | NULL |  | 元の表示テキスト（例: "1.2M"） |
| follower_count | INT UNSIGNED | NO |  |  | フォロワー数（正規化した数値） |
| created_at | DATETIME | NO | CURRENT_TIMESTAMP |  | レコード作成日時 |

## インデックス

| インデックス名 | 列名 | 種類 | 説明 |
|--------------|------|------|------|
| PRIMARY | id | PRIMARY | 主キー |
| uq_account_date | account_id, collection_date | UNIQUE | アカウント×日付で一意 |
| idx_account | account_id | INDEX | アカウント単位の検索用 |

## 関連テーブル

| このテーブルの列名 | テーブル名.列名 | 関連タイプ | 説明 |
|-----------|-----------|-----------|------|
| account_id | account_list.id | 多対一 | アカウントごとのフォロワー履歴 |

## 特徴
1. **日次スナップショット**
   - `UNIQUE (account_id, collection_date)` により、1アカウントあたり2日1レコードを保証。
2. **テキスト/数値の併存**
   - `follower_text`（生テキスト）と`follower_count`（正規化値）を保持し、可読性と分析の双方に対応。
3. **参照整合性**
   - `FOREIGN KEY (account_id) REFERENCES account_list(id) ON DELETE CASCADE ON UPDATE CASCADE` により、親削除時は履歴も連鎖削除。

## 備考
- 親に合わせて `account_id` は INT（符号あり）。
- 想定粒度は日次。重複投入時はユニーク制約違反となる。
- 文字列の桁表記（例: K/M）等は`follower_text`に保持し、数値化した結果は`follower_count`に保存する。
