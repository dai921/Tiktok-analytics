# products

## 概要
商品のマスターテーブル。TikTok動画で紹介・宣伝されている商品情報を管理する。

## テーブル定義

| 列名 | データ型 | NULL | デフォルト | インデックス | 説明 |
|-----|---------|------|----------|------------|------|
| id | BIGINT | NO | AUTO_INCREMENT | PRIMARY | 一意のID |
| name | VARCHAR(255) | NO | | UNIQUE | 商品名 |
| product_category | VARCHAR(255) | NO | - | 商品カテゴリー |
| created_at | DATETIME | NO | CURRENT_TIMESTAMP | | 登録日時 |
| updated_at | DATETIME | NO | CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | | 更新日時 |

## インデックス

| インデックス名 | 列名 | 種類 | 説明 |
|--------------|------|------|------|
| PRIMARY | id | PRIMARY | 主キー |
| idx_name | name | UNIQUE | 商品名の一意性を保証 |

## 関連テーブル

| このテーブルの列名 | テーブル名.列名 | 関連タイプ | 説明 |
|-----------|-----------|-----------|------|
| id | video_promotes_product.product_id | 多対多の中間テーブル | 各商品は複数の動画で紹介される可能性がある |

## 特徴

1. **商品情報の一元管理**
   - 商品名、カテゴリーなどの基本情報を管理

2. **柔軟な商品管理**
   - 商品の追加・変更が容易

## 変更点
1. **テーブル名やカラムの整理などの大きな変更**
   - 旧`product_master`テーブルから名称変更
   - 商品情報をより明確に表現

2. **データ構造の最適化**
   - 必要な情報のみを保持し、パフォーマンスを向上

## 備考
- 商品名は一意であるべきだが、同じ商品でもバリエーション（色・サイズなど）がある場合は、名前に区別情報を含める
